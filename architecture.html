01 · PROCESS

Claude Code

claude-code CLI

  • Fires hooks via ~/.claude/settings.json on every session event
  • Spawns hook-cli with event name + JSON payload on stdin
  • Blocks on PreToolUse until hook exits with allow / deny
  • Events: SessionStart · PreToolUse · PostToolUse · Stop

02 · BINARY

Hook Relay

hook-cli/src/main.rs

  • Reads event name + JSON payload from stdin
  • Windows: enriches payload with terminal HWND, sets Windows Terminal tab title via OSC-0
  • Connects to TCP bridge, sends BridgeEnvelope as newline-delimited JSON
  • For PreToolUse: blocks — races socket directive vs. console keypresses (y / n)

03 · TAURI / RUST

Bridge Server

src-tauri/src/bridge/

  • TCP server on Windows · Unix socket on Linux
  • Writes port to %APPDATA%/open-island/port — zero config for hook discovery
  • Session state machine: Idle → Running → Awaiting → Completed
  • Forwards events to frontend via Tauri emit()
  • resolve_permission command → unblocks hook via oneshot channel → directive over TCP

04 · SVELTE 5

Pill UI

src/App.svelte

  • 14px sliver at the top of the screen at rest — expands on hover or urgent session
  • Session list: cwd, phase, tool badge per active Claude Code session
  • Permission panel: Allow / Deny buttons wired to resolve_permission Tauri command
  • Click session row → AttachThreadInput + UIA tab select to focus the right terminal
